🕶️ Blind SQLi

A-02  ·  /advanced/blind-sqli

// Scenario

NovaCorp's user lookup API is injectable, but it only returns a binary response: "User found" or "User not found" — no data is ever echoed back. This is boolean-based blind SQL injection. By asking yes/no questions through crafted payloads (e.g., "Is the first character of the password 'a'?"), you can extract data one bit at a time. Use the built-in inference engine to recover the admin password hash.

🔍 User Lookup API
// Blind Inference Attack

The engine will automatically send boolean-based queries to extract the admin password character by character.

💡 Hint For blind SQLi, try admin' AND 1=1-- (user found) vs admin' AND 1=2-- (user not found). Use the automated extraction to demonstrate how data is recovered bit by bit.
