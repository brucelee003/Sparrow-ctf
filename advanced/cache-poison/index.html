🔴 Advanced 300 pts Cache Poisoning

☠️ Cache Poison

A-06  ·  /advanced/cache-poison

// Scenario

NovaCorp's CDN caches responses based on the URL path. However, the application reflects the X-Forwarded-Host header into the response (to build absolute URLs for SEO). The CDN doesn't strip this header before caching — so if you inject a malicious host, the cached response now contains a poisoned URL pointing to your server. Every subsequent user who receives the cached page is affected.

HTTP Headers
Custom Request
GET /home HTTP/1.1
Host: novacorp.com
X-Forwarded-Host:
Cache-Control: no-cache
