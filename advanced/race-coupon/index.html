⚡ Race Coupon

A-05  ·  /advanced/race-coupon

// Scenario

NovaCorp's checkout has a "one-use coupon" feature. The validation logic is: 1. Check if used → 2. Apply discount → 3. Mark as used. Steps 1 and 3 are non-atomic — there's a race window between the check and the mark. Send multiple simultaneous requests during that window and the coupon applies multiple times. Simulate a 5-request parallel burst and observe what happens.

🛒 NovaCorp Checkout
Cart Total
$100.00
Coupon Applied
None
One-time use coupon — $50 off
Vulnerable server logic:
// 1. Check
if (coupon.used) return error("Already used");
// 2. Apply (non-atomic gap ↓)
order.discount += coupon.value;
// 3. Mark used (too late!)
coupon.used = true;
💡 Hint Click the button to simulate 5 simultaneous coupon redemption requests. Because the check and mark are non-atomic, multiple requests pass the "already used?" check before any marks the coupon as used.
