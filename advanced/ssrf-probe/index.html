🌐 SSRF Probe

A-01  ·  /advanced/ssrf-probe

// Scenario

NovaCorp's "URL Preview" feature fetches a user-supplied URL on the server side and renders a preview. The server sits inside the internal network and can reach services that aren't exposed to the internet — like the metadata service (http://169.254.169.254/) or internal APIs (http://internal.novacorp.local/). Make the server fetch an internal resource.

// Internal Targets (server-side reachable)
URL Description
http://169.254.169.254/latest/meta-data/ AWS Instance Metadata — contains IAM credentials
http://internal.novacorp.local/ Internal admin API — returns flag
http://localhost:8080/admin Internal admin service on localhost
🔗 URL Preview Service
💡 Hint The server fetches the URL you provide. Try internal addresses: http://169.254.169.254/ (AWS metadata) or http://internal.novacorp.local/. These are unreachable from the internet but accessible from the server's internal network.
