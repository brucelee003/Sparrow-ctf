💾 Easy Backup

B-03  ·  /beginner/easy-backup

// Scenario

A NovaCorp developer edited config.php directly on the production server using the nano text editor. Nano silently creates a backup copy named config.php~ in the same directory — a standard Unix text-editor behavior. That backup file contains the original config including database credentials and a secret token. It's sitting there, world-readable, waiting to be fetched.

// Endpoints
Method URL Description
GET config.php Normal config (redacted output)
GET config.php~ Nano backup file — contains credentials + flag
📁 File Browser — /beginner/easy-backup/

Try accessing these files directly in your browser address bar:

📄 config.php 📄 config.php~  (backup file)
// Steps
1
Notice the config.php file is accessible
Open config.php — it shows a redacted/empty response
2
Try the backup filename convention
Append ~ to the URL: config.php~
3
Read the backup file contents
The backup contains unredacted config including the flag
💡 Hint nano, vi, and other editors create backup files with specific suffixes. Common ones: ~ (nano), .bak, .old, .orig.
