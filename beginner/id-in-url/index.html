🔢 ID in URL

B-04  ·  /beginner/id-in-url

// Scenario

NovaCorp's user profile page loads data based on a numeric ID in the URL query string: ?id=1. There's no server-side check to verify that you're actually allowed to view a given ID. You've been given a normal user account (ID 1). The admin account is ID 2. Can you access the admin's private profile data?

// Endpoints
Method URL Description
GET index.html?id=1 Your profile (normal user)
GET index.html?id=2 Admin profile — contains flag
👤 User Profile Viewer ?id=1
💡 Hint Look at the URL bar. The ?id= parameter controls which profile loads. Try changing the number. This is called an Insecure Direct Object Reference (IDOR).
