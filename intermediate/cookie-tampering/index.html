🍪 Cookie Tampering

I-03  ·  /intermediate/cookie-tampering

// Scenario

After login, NovaCorp sets a cookie: role=user. The server reads this cookie on every request to determine permissions — without any signature or verification. Whoever controls the cookie, controls the role. Change it to role=admin using DevTools → Application → Cookies, then access the admin panel.

// Steps
1
Log in as a normal user
Click the button below — it sets a role=user cookie in your browser
2
Open DevTools → Application → Cookies
Find the nova_role cookie for this page
3
Edit the cookie value
Change value from user to admin
4
Click "Load Admin Panel"
The page reads the cookie and grants access if role=admin

🚩 Admin Panel — Access Granted

You modified the nova_role=admin cookie and gained admin access. The server never verified the cookie's authenticity — no HMAC, no signing, nothing.
