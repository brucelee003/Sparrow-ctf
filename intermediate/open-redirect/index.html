↪️ Open Redirect

I-07  ·  /intermediate/open-redirect

// Scenario

NovaCorp's login page redirects users post-authentication using a ?next= URL parameter. The server doesn't validate whether the target is on the NovaCorp domain — it redirects anywhere. This can be used for phishing: send a victim a link like novacorp.com/login?next=https://evil.com and they'll be sent to the attacker's site after logging in (or just by visiting the URL, in this simulation).

// Endpoint
URL Effect
?next=/dashboard Safe internal redirect
?next=https://evil.com Open redirect to external site (vulnerable)
?next=https://attacker.novacorp-evil.com Bypass with subdomain confusion
