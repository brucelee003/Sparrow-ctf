🗂️ Path Traversal Lite

I-08  ·  /intermediate/path-traversal-lite

// Scenario

NovaCorp's document viewer loads files based on a ?file= parameter. The application is supposed to only serve files from /docs/, but doesn't sanitize the filename. Using ../ sequences, you can "traverse" out of the docs directory and read files from other parts of the server — including sensitive configuration files.

// Endpoint
URL Resolves To Result
?file=readme.txt /docs/readme.txt Normal doc
?file=../../../etc/passwd /etc/passwd Server file (flag inside)
?file=../../config/secret.cfg /config/secret.cfg Config file (flag)
📄 NovaCorp Document Viewer
💡 Hint Try ?file=../../../etc/passwd or ../../config/secret.cfg to traverse outside the /docs/ directory.
