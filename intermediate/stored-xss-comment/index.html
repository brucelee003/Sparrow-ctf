💾 Stored XSS Comment

I-02  ·  /intermediate/stored-xss-comment

// Scenario

NovaCorp's tech blog accepts public comments. The backend stores submissions directly without any HTML encoding, then re-renders them on load. When the admin periodically reviews the comment queue, any stored script executes in their browser context. You post — they get hit. Submit a malicious comment, then use the "Admin View" to simulate what the admin sees.
