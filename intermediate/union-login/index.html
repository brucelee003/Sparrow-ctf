💉 Union Login

I-01  ·  /intermediate/union-login

// Scenario

NovaCorp's login endpoint constructs its database query using direct string concatenation: SELECT * FROM users WHERE username='[input]' AND password='[input]'. No parameterized queries, no prepared statements. A single quote in your input breaks out of the string literal and lets you rewrite the logic of the query itself. Bypass authentication entirely — no valid password needed.

// Backend Query (vulnerable)
login.php (server-side)
$query = "SELECT * FROM users WHERE username='" . $_POST['user'] . "' AND password='" . $_POST['pass'] . "'";
// No sanitization. String concatenation. Classic.
🔐 NovaCorp Member Login

You bypassed authentication using SQL injection. The query evaluated to TRUE for all rows.
